**CompanyService — adicionar `createForRegistration` (após o método `findByCnpj` existente):**

```dart
  /// Cria uma empresa via RPC SECURITY DEFINER, contornando RLS.
  ///
  /// Deve ser chamado ANTES de [AuthService.signUp] no fluxo de self-registration,
  /// pois o usuário ainda não está autenticado neste ponto.
  ///
  /// Retorna o UUID (String) da empresa criada. Lança [PostgrestException] se:
  ///   - CNPJ já cadastrado ('CNPJ já cadastrado no sistema')
  ///   - CNPJ inválido ('CNPJ deve ter 14 dígitos')
  ///   - Nome vazio ('Nome da empresa é obrigatório')
  ///
  /// Edge case: se [signUp] falhar após este método ter sucesso, a empresa
  /// existirá sem owner. Uma nova tentativa com o mesmo CNPJ retornará o erro
  /// de CNPJ duplicado. Limpeza de órfãos é responsabilidade do superuser (v1.4+).
  Future<String> createForRegistration({
    required String cnpj,
    required String name,
  }) async {
    final companyId = await _client.rpc(
      'create_company_for_registration',
      params: {
        'p_cnpj': cnpj,
        'p_name': name,
      },
    );
    return companyId as String;
  }
```
